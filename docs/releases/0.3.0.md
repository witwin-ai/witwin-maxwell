# WiTwin Maxwell 0.3.0

Maxwell 0.3.0 is a major expansion of the PyTorch-native, GPU-first electromagnetic workflow. It adds native CUDA adjoint coverage, broader material composition, broadband/Bloch TFSF sources, mature near-to-far processing, expanded Tidy3D interoperability, and a physically audited cross-solver validation path.

## Highlights

- Native CUDA FDTD adjoint reverse kernels for standard/CPML/Bloch fields and supported anisotropic, conductive, dispersive, nonlinear, TFSF, and multi-source scenes.
- Full off-diagonal electric anisotropy, 2D sheets and Graphene, SIBC lossy metal, custom dispersion, perturbation, nonlinearity, and space-time modulation.
- Broadband and arbitrary-axis grating TFSF, plus broader lossy/anisotropic/bent mode-source and port workflows.
- Background-aware and curved-surface equivalent-current propagation, directivity, bistatic RCS, and expanded S-parameter processing.
- Stable-ABI CUDA wheels for Linux and Windows, validated across Python 3.10-3.14 and PyTorch 2.10-2.12.
- Tidy3D export and validation fixes for physical source amplitudes, Bloch units, external PML, monitor components, SIBC placement, complex phase, flux apertures, and solver-independent scalar observables.
- Native CUDA SIBC and anisotropic-conduction capture, plus device-resident modulation timing for CUDA Graph replay.

## FDFD maturity notice

`Simulation.fdfd(...)` is available through the same `Scene -> Simulation -> Result` architecture, but it remains a limited/experimental CUDA-only single-frequency solver. It currently targets supported linear isotropic or diagonal-electric scenes with `none`/PML boundaries. Use FDTD for nonuniform grids, magnetic or nonlinear media, full off-diagonal anisotropy, in-domain PEC, symmetry, and the broader source/monitor/adjoint feature set.

## Numerical validation

The Maxwell-vs-Tidy3D campaign now compares common physical-domain coordinates with external PML, coordinate-aligned complex field slices, visual phase/boundary diagnostics, and physical scalar observables. It avoids fitted amplitudes and removes only a single unit-magnitude source-reference phase where physically justified.

The complete conventions, encountered pitfalls, fixes, representative Group 7 results, and intentionally unresolved operator-level residuals are documented in [Maxwell-Tidy3D Numerical Alignment for 0.3.0](https://github.com/witwin-ai/witwin-maxwell/blob/witwin-maxwell-v0.3.0/docs/validation/tidy3d-numerical-alignment-0.3.0.md).

The release audit found no new Torch or CPU computation in the functional FDTD time-step hot path. Warm native-CUDA timings on the release RTX 5080 were 0.483 ms/step (`dipole_vacuum`), 0.425 ms/step (`planewave_vacuum`), and 4.232 ms/step for native adjoint backward, all slightly faster than the recorded baseline. FDFD tests and FDFD-vs-FDTD comparisons were intentionally not used as release gates because FDFD is not feature-complete.
